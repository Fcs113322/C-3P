# 钢桩桩位偏差计算

## 功能

- 导入测量文件并自动识别三点桩边点。
- 三点确定圆，计算实际桩中心。
- 导入或手动维护理论桩位。
- 高级设置“桩号对应”：当测量桩号经过精简、与理论桩号不一致时，可上传对应表并选择两列建立映射。
- 左侧设置区使用标签页切换，减少页面拥挤。
- 右侧绘图区和数据区之间支持拖动调整大小。
- 绘图画布固定尺寸并居中显示，不随窗口大小变化而缩放。
- 图例位于绘图窗口右下角，按多行显示。
- 图例使用对应颜色和线型标识。
- 文件选择入口使用醒目的虚线按钮。
- 理论桩位支持点击“添加”弹窗录入，点击已保存桩位可弹窗编辑。
- 理论桩位支持一键删除全部已保存数据。
- 标题后方带有 `Fcs_×TREA` 标识和 GitHub 图标。
- GitHub 图标可点击跳转到 GitHub 主页。
- 状态列使用柔和颜色标记，`超限` 会以低饱和红色突出显示。
- 侧边栏按钮和文件选择入口统一宽度与蓝色系视觉风格。
- 图上显示实际桩位、理论桩位、偏位方向和 `平面偏差(mm)`。
- 表格显示 `ΔX(mm)`、`ΔY(mm)`、`平面偏差(mm)`。
- 偏差表格内容居中，默认列宽更紧凑，列宽可拖动调整。
- 当 `平面偏差(mm) > 100` 时，状态显示 `超限`。
- 导出 Excel 结果。

## 支持文件

测量文件支持：

- `.xlsx`
- `.xlsm`
- `.csv`
- `.txt`

测量点名格式：

```text
日期_桩名称_点序号
```

示例：

```text
20260705_mp63-4_1
20260705_mp63-4_2
20260705_mp63-4_3
```

理论桩位文件按列读取：

```text
桩号、X、Y、H
```

其中 `H` 可省略，只要确保前三列为 `桩号、X、Y`。

## 桩号对应

当测量文件中的桩号与理论文件中的桩号不一致时，可以使用“高级设置：桩号对应”。该区域在“对应”标签页中默认展开。

对应文件支持：

- `.xlsx`
- `.xlsm`
- `.csv`
- `.txt`

对应文件至少需要两列，可以有标题行。上传后页面会自动识别出列，并让你选择：

- 测量桩号列：测量文件中实际出现的简化桩号。
- 展示/理论桩号列：最终展示、匹配理论桩位和导出结果使用的正确桩号。

启用后：

- 测量文件仍按原始测量桩号自动分组三点。
- 结果表、图上标签和导出 Excel 使用对应后的展示桩号。
- 理论桩位列表和理论桩位匹配都会实时使用对应后的展示桩号；如果理论库中保存的是简化桩号，也会通过对应关系参与匹配。

## 注意

- 理论桩位数据保存在当前浏览器的本地存储中，不会上传到服务器。
- 桩号对应关系也保存在当前浏览器的本地存储中。
- 换电脑、换浏览器或清理浏览器数据后，理论桩位需要重新导入。
- Excel 读取和导出使用浏览器端 SheetJS 库，通过 CDN 加载。
